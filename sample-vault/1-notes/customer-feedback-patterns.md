# Customer Feedback Patterns

Three recurring themes show up in interviews:

- people trust notes that stay close to plain markdown
- they want tasks to surface quietly instead of becoming a second project tool
- AI is welcome when it drafts, but not when it surprises

Implications:

- keep writeback opt-in
- make note provenance obvious
- connect AI actions to the source note every time
